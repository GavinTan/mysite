iis部署django flask​

手动添加

  1. FastCGI设置-添加应用程序

    完整路径：d:\soft\python\python38-32\python.exe（python.exe的绝对路径）

    参数：d:\soft\python\python38-32\lib\site-packages\wfastcgi.py（wfastcgi.py的绝对路径）

    环境变量：

    • PYTHONPATH=C:\Users\gavin\Desktop\mytest
    • WSGI_HANDLER=mytest.wsgi.application（flask为filename.app）
    • DJANGO_SETTINGS_MODULE=mytest.settings
    • DJANGO_SETTINGS_MODULE=mytest.settings

  2. 网站-处理程序映射-添加模块映射

    请求路径：*

    模块：FastCgiModule

    可执行文件：d:\soft\python\python38-32\python.exe|d:\soft\python\python38-32\lib\site-packages\wfastcgi.py（wfastcgi-enable输出的内容）

    名称：DjangoFastCGI

  3. 给项目目录添加IIS_USERS用户组权限（没有权限会出现500错误）

  4. 静态文件（static）

    1. django生成static（python manage.py collectstatic）文件夹

    2. 控制面板-添加功能-internet 信息服务-万维网服务-常见HTTP功能-静态内容

    3. 右键网站-添加虚拟目录

      别名：static（对应静态文件目录）

      路径：C:\Users\gavin\Desktop\mytest\static（静态文件路径）

    4. 点击添加的虚拟目录-处理映射程序-删除上面添加的DjangoFastCGI模块

通过配置web.config

  1. 安装 IIS
  2. 同时在万维网服务应用程序开发功能里勾选上CGI、ASP.NET（使用web.config文件在iis里处理程序映射会提示错误）
  3. 在管理员cmd里执行wfastcgi-enable（默认会自动生成iis中的FastCGI设置）
  4. 添加网站设置项目目录并在目录下配置web.config文件
  5. 给项目目录添加IIS_USERS用户组权限
  6. 静态文件（static）
    1. django生成static（python manage.py collectstatic）文件夹
    2. iis配置虚拟目录指定路径为 django生成static的目录
    3. 在static目录下添加web.config静态配置文件（iis静态文件模块需要在功能里安装上万维网服务-常见HTTP功能勾选上静态内容）

dism系统修复​

扫描全部系统文件和系统映像文件是否与官方版一致（无论你是否备份或制作过系统映像）

Dism /Online /Cleanup-Image /ScanHealth

如果出现错误提示就需要检查文件的损坏程度，结果一般有两种：一般性损坏可以修复和情况严重完全无法修复。

Dism /Online /Cleanup-Image /CheckHealth

是前者的话联网将系统映像文件中与官方文件不相同的文件还原成官方系统的源文件，系统中安装的软件和用户设置保留不变，比重装系统要好得多，而且在修复时系统未损坏部分可以正常运行，不耽误大家手头上的事情。检查结果是后者的话，除了重装系统外，没有太好的办法解决。

DISM /Online /Cleanup-image /RestoreHealth

没有网络或怕它占用网络资源，也可以启动本地修复命令：

Dism /Online /Cleanup-Image /RestoreHealth /Source:c:\test\mount\windows /LimitAccess

检查映像版本：

Dism /online /Get-CurrentEdition

检查系统是否可升级：

Dism /online /Get-TargetEditions

参考：https://www.20hn.com/146.html

clash 端口被占用​

显示 TCP 的排除范围

netsh interface ipv4 show excludedportrange protocol=tcp

由于hyper-v保留了这些端口，所以需要先关闭hyper-v，保留想要保留的端口，再开启hyper-v功能

关闭hyper-v：

dism.exe /Online /Disable-Feature:Microsoft-Hyper-V

添加保留端口：

netsh int ipv4 add excludedportrange protocol=tcp startport=1080 numberofports=2
netsh int ipv4 add excludedportrange protocol=tcp startport=1099 numberofports=1
netsh int ipv4 add excludedportrange protocol=tcp startport=3306 numberofports=1

启用hyper-v：

dism.exe /Online /Enable-Feature:Microsoft-Hyper-V /A

  • 家庭版：供家庭用户使用，无法加入Active Directory和Azure AD，不允许远程链接；
  • 专业版：供小型企业使用，在家庭版基础上增加了域账号加入、bitlocker、企业商店等功能；
  • 企业版：供中大型企业使用，在专业版基础上增加了DirectAccess，AppLocker等高级企业功能；
  • 教育版：供学校使用 (学校职员、管理人员、老师和学生) 其功能几乎和企业版一样，针对学校授权而已；
  • LTSB版：无Edge浏览器、小娜，无磁铁，可选是否下载和安装补丁，其它版都不能自选补丁，适合办公；
